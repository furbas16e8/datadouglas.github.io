Limpeza e pr-processamento

A limpeza e o pr-processamento de dados, incluindo a harmonizao de categorias no documentadas ou inconsistentes, so etapas que podem consumir uma poro significativa do tempo de um projeto, mas so essenciais para a validade e robustez dos modelos subsequentes (James et al., 2021).

Para fazer uma inspeo inicial nos dados, utilizo o comando glimpse():

In[]:
# Exibie a estrutura do dataframe para entender os tipos de dados de cada coluna

Aqui possvel perceber que as variveis sex, education e marriage foram interpretadas como variveis numricas (int). Mas de acordo com a descrio das variveis fornecidas pelo dataset so variveis categricas.

Ento necessrio converter as colunas correspondentes a cada uma para o tipo factor.

Aqui, pude perceber que as colunas EDUCATION e MARRIAGE apresentam um valor (0) que no est informado nas definies das variveis do dataset disponibilizado pelo Kaggle.

Segundo a plataforma:

  • MARRIAGE: Estado civil (1=casado, 2=solteiro, 3=outros)
  • EDUCATION: Escolaridade (1=ps-graduao, 2=universidade, 3=ensino mdio, 4=outros, 5=desconhecido, 6=desconhecido)

Ou seja, no temos uma definio para o valor (0) que apresenta 14 observaes em EDUCATION E 54 EM MARRIAGE. Decidi agrupar esse valor na categoria Unknown em EDUCATION e em Others em MARRIAGE para facilitar a anlise.

Alm disso, seguindo o exposto por (James et al., 2021), para melhorar a interpretabilidade das variveis categricas, decidi recodific-las (incluindo a varivel default.payment.next.month) para evitar a consulta contnua descrio de cada uma.

credit_card_df <- credit_card_df %>%

                              `0` = "Unknown",

                              `2` = "University",
                              `3` = "High School",
                              `4` = "Others",
                              `5` = "Unknown",
                              `6` = "Unknown"),
    MARRIAGE = recode_factor(MARRIAGE, `0` = "Other", `1` = "Married", `2` = "Single", `3` = "Other"),

    default.payment.next.month = recode_factor(default.payment.next.month, `0` = "No", `1` = "Yes")
  ) %>%
  # Renomeia a varivel alvo
  rename(DEFAULT = default.payment.next.month)

Dado o resultado do cdigo acima, agora sabemos que o dataset no possui lacunas nos dados. Por outro lado, a varivel education possui valores como "Others" e "Unknown". Diferentemente de lacunas ou de clulas preenchidas com "NA", tais valores no so buracos no dataset.

  • Others (Outros): informa que, quando um cliente respondeu ao questionrio para iniciar um contrato com o banco de Taiwan, seu nvel de escolaridade no estava listado nas opes possveis.
  • Unknown (Desconhecido): por sua vez nos diz que a escolaridade do cliente , lietralmente, desconhecida.

Um exemplo mais comum acontece quando em um formulrio, na categoria "Sexualidade" as opes selecionveis podem ser: "heteressexual", "homossexual", "bissexual", "outro (especificar)" e "Prefiro no responder".

Dessa forma o dado existe, e a importncia dele est no padro e ausncia. Se os clientes escolhem "Outros" ou "Prefiro no responder" o fazem por algum motivo que pode se correlacionar com a inadimplncia. Ento, mesmo que sozinhos esses valores no possam ser possveis de interpretao, dado todo o conjunto do dataset essas categorias em si so uma informao valisosa. Apag-las sem algum critrio apenas para simplificar os dados causaria vis e possivelmente colocaria toda a anlise em xeque.

Considerando o valor 0 como uso de crdito rotativo ou seja, de no inandimplncia, podemos juntar esses valores com o valor 1 representanto todos os clientes no-inadimplentes.

In[12]:
# Seleciona as colunas de histrico de pagamento
pay_cols <- c("PAY_0", "PAY_2", "PAY_3", "PAY_4", "PAY_5", "PAY_6")

# Recodifica as colunas: valores <= 0 so "Adimplente", > 0 so "Delinquent"
credit_card_df_cleaned <- credit_card_df %>%
  mutate(across(all_of(pay_cols), 
                ~factor(if_else(.x <= 0, "Adimplente", "Inadimplente"))))

# Exibe a contagem das novas categorias para PAY_0 para verificar a transformao

Esta tcnica de agrupar mltiplos valores em categorias menores uma forma de discretizao. Ela pode simplificar o modelo, reduzir o rudo e capturar relaes no-lineares de forma mais eficaz, especialmente quando h um limiar claro entre comportamentos, como inadimplente vs. no-inadimplente (Hastie et al., 2009).
